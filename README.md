<p>yolo格式的标签为txt格式的文件，文件名跟对应的图片名一样，除了后缀改为了.txt。
具体格式如下：</p>
<ul>
<li>每个目标一行，整个图片没有目标的话不需要有txt文件</li>
<li>每行的格式为<code>class_num x_center y_center width height</code></li>
<li>其中<code>class_num</code>取值为<code>0</code>至<code>total_class - 1</code>，框的四个值<code>x_center</code> <code>y_center</code> <code>width</code> <code>height</code>是相对于图片分辨率大小正则化的<code>0-1</code>之间的数，左上角为<code>(0,0)</code>，右下角为<code>(1,1)</code></li>
</ul>

<p>有参：</p>
<ul>
<li><code>--weights</code> (<g-emoji class="g-emoji" alias="star" fallback-src="https://github.githubassets.com/images/icons/emoji/unicode/2b50.png">⭐</g-emoji>)指定权重，如果不加此参数会默认使用COCO预训的<code>yolov5s.pt</code>，<code>--weights ''</code>则会随机初始化权重</li>
<li><code>--cfg</code> 指定模型文件</li>
<li><code>--data</code> (<g-emoji class="g-emoji" alias="star" fallback-src="https://github.githubassets.com/images/icons/emoji/unicode/2b50.png">⭐</g-emoji>)指定数据文件</li>
<li><code>--hyp</code>指定超参数文件</li>
<li><code>--epochs</code> (<g-emoji class="g-emoji" alias="star" fallback-src="https://github.githubassets.com/images/icons/emoji/unicode/2b50.png">⭐</g-emoji>)指定epoch数，默认300</li>
<li><code>--batch-size</code> (<g-emoji class="g-emoji" alias="star" fallback-src="https://github.githubassets.com/images/icons/emoji/unicode/2b50.png">⭐</g-emoji>)指定batch大小，默认<code>16</code>，官方推荐越大越好，用你GPU能承受最大的<code>batch size</code>，可简写为<code>--batch</code></li>
<li><code>--img-size</code> 指定训练图片大小，默认<code>640</code>，可简写为<code>--img</code></li>
<li><code>--name</code> 指定结果文件名，默认<code>result.txt</code></li>
<li><code>--device</code> 指定训练设备，如<code>--device 0,1,2,3</code></li>
<li><code>--local_rank</code> 分布式训练参数，不要自己修改！</li>
<li><code>--log-imgs</code> W&amp;B的图片数量，默认16，最大100</li>
<li><code>--workers</code> 指定dataloader的workers数量，默认<code>8</code></li>
<li><code>--project</code> 训练结果存放目录，默认./runs/train/</li>
<li><code>--name</code> 训练结果存放名，默认exp</li>
</ul>
<p>无参：</p>
<ul>
<li><code>--rect</code>矩形训练</li>
<li><code>--resume</code> 继续训练，默认从最后一次训练继续</li>
<li><code>--nosave</code> 训练中途不存储模型，只存最后一个checkpoint</li>
<li><code>--notest</code> 训练中途不在验证集上测试，训练完毕再测试</li>
<li><code>--noautoanchor</code> 关闭自动锚点检测</li>
<li><code>--evolve</code>超参数演变</li>
<li><code>--bucket</code>使用gsutil bucket</li>
<li><code>--cache-images</code> 使用缓存图片训练</li>
<li><code>--image-weights</code> 训练中对图片加权重</li>
<li><code>--multi-scale</code> 训练图片大小+/-50%变换</li>
<li><code>--single-cls</code> 单类训练</li>
<li><code>--adam</code> 使用torch.optim.Adam()优化器</li>
<li><code>--sync-bn</code> 使用SyncBatchNorm，只在分布式训练可用</li>
<li><code>--log-artifacts</code> 输出artifacts,即模型效果</li>
<li><code>--exist-ok</code> 如训练结果存放路径重名，不覆盖已存在的文件夹</li>
<li><code>--quad</code> 使用四分dataloader</li>
</ul>

<h4><a id="user-content-2-自定义检测" class="anchor" aria-hidden="true" href="#2-自定义检测"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>2. 自定义检测</h4>
<p>使用权重<code>./weights/yolov5s.pt</code>去推理<code>./data/images</code>文件夹下的所有媒体，并且推理置信度阈值设为0.5:</p>
<div class="highlight highlight-source-shell"><pre>$ python detect.py --source ./data/images/ --weights ./weights/yolov5s.pt --conf 0.5</pre></div>
<h4><a id="user-content-3-检测指令说明" class="anchor" aria-hidden="true" href="#3-检测指令说明"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>3. 检测指令说明</h4>
<p>自己根据需要加各种指令。</p>
<p>有参：</p>
<ul>
<li><code>--source(⭐)</code>  指定检测来源</li>
<li><code>--weights</code> 指定权重，不指定的话会使用yolov5s.pt预训练权重</li>
<li><code>--img-size</code> 指定推理图片分辨率，默认640，也可使用<code>--img</code></li>
<li><code>--conf-thres</code> 指定置信度阈值，默认0.4，也可使用<code>--conf</code></li>
<li><code>--iou-thres</code> 指定NMS(非极大值抑制)的IOU阈值，默认0.5</li>
<li><code>--device</code> 指定设备，如<code>--device 0</code> <code>--device 0,1,2,3</code> <code>--device cpu</code></li>
<li><code>--classes</code> 只检测特定的类，如<code>--classes 0 2 4 6 8</code></li>
<li><code>--project</code> 指定结果存放路径，默认./runs/detect/</li>
<li><code>--name</code> 指定结果存放名,默认exp</li>
</ul>
<p>无参：</p>
<ul>
<li><code>--view-img</code> 图片形式显示结果</li>
<li><code>--save-txt</code> 输出标签结果(yolo格式)</li>
<li><code>--save-conf</code> 在输出标签结果txt中同样写入每个目标的置信度</li>
<li><code>--agnostic-nms</code> 使用agnostic NMS(前背景)</li>
<li><code>--augment</code> 增强识别，速度会慢不少。<a href="https://github.com/ultralytics/yolov5/issues/303">详情</a></li>
<li><code>--update</code> 更新所有模型</li>
<li><code>--exist-ok</code> 若重名不覆盖</li>
</ul>
<h2><a id="user-content-测试" class="anchor" aria-hidden="true" href="#测试"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>测试</h2>
<h4><a id="user-content-1测试命令" class="anchor" aria-hidden="true" href="#1测试命令"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>1.测试命令</h4>
<p>首先明确，推理是直接检测图片，而测试是需要图片有相应的真实标签的，相当于检测图片后再把推理标签和真实标签做mAP计算。<br>
使用<code>./weights/yolov5x.pt</code>权重检测<code>./data/coco.yaml</code>里定义的测试集，图片分辨率设为672。</p>
<div class="highlight highlight-source-shell"><pre>$ python test.py --weights ./weights/yolov5x.pt --data ./data/coco.yaml --img 672</pre></div>
<h4><a id="user-content-2各指令说明" class="anchor" aria-hidden="true" href="#2各指令说明"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>2.各指令说明</h4>
<p>有参：</p>
<ul>
<li><code>--weights(⭐)</code> 测试所用权重，默认yolov5sCOCO预训练权重模型</li>
<li><code>--data(⭐)</code> 测试所用的.yaml文件，默认使用<code>./data/coco128.yaml</code></li>
<li><code>--batch-size</code> 测试用的batch大小，默认32，这个大小对结果无影响</li>
<li><code>--img-size</code> 测试集分辨率大小，默认640，测试建议使用更高分辨率</li>
<li><code>--conf-thres</code>目标置信度阈值，默认0.001</li>
<li><code>--iou-thres</code>NMS的IOU阈值，默认0.65</li>
<li><code>--task</code> 指定任务模式，train, val, 或者test,测试的话用<code>--task test</code></li>
<li><code>--device</code> 指定设备，如<code>--device 0</code> <code>--device 0,1,2,3</code> <code>--device cpu</code></li>
<li><code>--project</code> 指定结果存放路径，默认./runs/test/</li>
<li><code>--name</code> 指定结果存放名,默认exp</li>
</ul>
<p>无参：</p>
<ul>
<li><code>--single-cls</code> 视为只有一类</li>
<li><code>--augment</code> 增强识别</li>
<li><code>--verbose</code> 输出各个类别的mAP</li>
<li><code>--save-txt</code> 输出标签结果(yolo格式)</li>
<li><code>--save-hybrid</code> 输出标签+预测混合结果到txt</li>
<li><code>--save-conf</code> 保存置信度至输出文件中</li>
<li><code>--save-json</code> 保存结果为json</li>
<li><code>--exist-ok</code> 若重名不覆盖</li>
</ul>
